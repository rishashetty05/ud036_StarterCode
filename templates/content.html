Introduction To Netwroks

In Information technology, a netwok is a series of points or nodes interconnected by communication paths. Network can interconnect with other networks and also contain subnetwork. Though not directly connected, communication and exchange of information is sill possible in a network. This is a general definition, as there is a lot of versatality to networks. Internet too is a type of Network, Computer Network specifically. The Internet is more of a concept than an actual tangible entity, and it relies on a physical infrastructure that connects networks to other networks.


We can always measure a network based on two ways : 1. Latency 2. Bandwidth.
The concept of Latency and Bandwidth.

Computer networks or data network differ in the transmission media used to carry their signals, the communications protocols to organize network traffic, the network's size i.e. the number of bytes in a data-segment(MTU), topology and organizational intent. In most cases, communications protocols are layered on (i.e. work using) other more specific or more general communications protocols, except for the physical layer that directly deals with the transmission media. Servers

At the most basic level, when we type in an addrss,(Uniform Resource Locator) say "http://www.example.com/foo?p=1/sample.gif", and we see a page on our Computer screen. Behind the scenes, the browser actually breaks the URL into :

  1. PROTOCOL - HYper Text Transfer Protocol - HTTP
  2. Server Name - Domain Name (Host)
  3. File Name (Path along with file name)
The Server name (www.example.com)translates to an IP ADDRESS which our browser connects with, and directs through path (foo?p=1/) to get the file name (sample.gif). HTTP is the protocol or language used to make a request in the form of a Query Parameter which mostly a GET Request asking for the given file name. The Server machine makes its service available through numbered path, usually visible at the end pf the URL called port, eg. http://www.xxxx.yyyy.com:918. When no port number is specifiec we assume its the defult port: 80. AT times, ther are extra FRAGMENTS usually seperated by teh "#" sign.Fragments are used to particularly refernce a Page we are looking at. In the URL, it comes after query parameter.

String Sustitution & Advanced String Substitution

String Substitution isused to sustitute a variable form data which has to be reused and generated every time while coding. For eg. 

                        "some bold text"
If we had to use the above string everywhere everytime, it would be difficult task. Instead we can use string substitution to reuse the 
                        "%s" % VARIABLE
Now we can sustitute the variable to which ever value we need. Similary if its multiple variable sustitution can be done as below 
                        "some text %s then some text %s" %(variable1, variable2)
A sample on using string substitution. In this example, we are trying to use write_form which takes two parameters, self and error. But notice that it says error="". This is an example of an optional parameter. It means that you can specify a value for the error parameter when you call the function but you don't have to. If you don't specify a value, it will take a default value (in this case, the empty string "").

What happens if you want to include the same variable twice in the same string? We can do this by including a name identifier in parentheses: 

                        "text %(NAME)s text" % {"NAME": value}
